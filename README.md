請扮演資深前端開發工程師，協助我從零開始打造一個網頁版記帳網站。
【限制條件】：
1. 嚴禁使用任何前端框架（例如 React, Vue, Angular）。
2. 請使用純 HTML5、CSS3（可使用 Tailwind CSS CDN 加快切換樣式）與純 JavaScript (Vanilla JS) 實作。
3. 資料儲存請先使用瀏覽器的 localStorage，不需連接後端資料庫。
4. 圖表分析請引入 Chart.js (CDN) 來繪製。

【網站規格與功能需求】：
1. 畫面配置（單頁式網頁 SPA）：
   - 儀表板 (Dashboard)：顯示本月總收入、總支出、剩餘預算、一幅支出分類圓餅圖，以及一幅每月收支趨勢折線圖。
   - 記帳表單 (Form)：包含日期選擇器、流向切換（收入/支出）、分類下拉選單、金額輸入框（僅限數字）、備註文字欄、儲存按鈕。
   - 歷史清單 (History)：按日期倒序排列的流水帳列表，每筆紀錄需顯示日期、分類、備註、金額，並附帶「刪除」按鈕。
   - 理財建議區 (Insights)：根據現有數據給予一句理財文字提示。

2. 分類項目：
   - 支出分類：三餐、飲品、點心、交通、購物、娛樂、日用品、房租、醫療、禮物、數位、生活、其他。
   - 收入分類：薪資、獎金、投資、零用錢、其他。

3. 邏輯與分析功能：
   - 可以切換「日/週/月」視角查看歷史紀錄。
   - 設定每月總預算功能：當月總支出大於預算 80% 時，儀表板的預算進度條顯示黃色；超過 100% 顯示紅色。
   - 智能理財邏輯：若「飲品+點心」佔總支出超過 15%，理財建議區需提示「減少手搖飲與下午茶，可省下更多錢」；若總支出大於總收入，提示「本月入不敷出，請調降購物與娛樂預算」。
